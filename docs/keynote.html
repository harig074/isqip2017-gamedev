Game engines


  • Abstracts technicalities and lets developers focus on game elements
  • Provides reusable components - rendering, audio, physics etc.
  • 2d and 3d engines
  • Design first vs Code first
  • Popular 2d game engines
    • - Unity 3d (Free with restrictions)
    • - Unreal Engine (Free with royalty)
    • - Phaser (HTML5)
    • - Cocos2d family
    • - Gamemaker Studio
    • - Godot Engine

Godot Engine


  • Initially closed-source - used by many Latin American game studios
  • Now free and open source - MIT License
  • 2d and 3d Game engine
  • Design first approach
  • Integrated IDE, Physics and other game capabilities
  • Rich features
  • Active community
  • Program in GDScript which is similar to Python
  • Use C++ when performance is important

Scripting


  • Scripring allows programmatic access to scenes and nodes
  • Build complex and dynamic logic and control game world in finest detail possible
  • Add new nodes to world, check interaction between them, respond to user input
  • Interact with servers, hardware devices
  • GDScript is a simple dynamically typed language, similar to Python
  • Developed from scratch by Godot team
  • Heavily optimized for game development
  • At the expense of convenience and ease of use, there is a performance penalty
  • Most of the heavy lifting is done by the Engine, leaving game logic to scripts
  • For more computation intensive code, use C++

GDSCript


  • Class name is file name. Recommended to keep same as root node name
  • Extends root node type - Follows multi-level inheritance
  • Object oriented design is encouraged
  • Functions can only be written inside classes
  • Dynamic types
  • Object oriented design is encouraged
  • Heavily optimized for game development
  • At the expense of convenience and ease of use, there is a performance penalty
  • Most of the heavy lifting is done by the Engine, leaving game logic to scripts
  • For more computation intensive code, use C++

Physics


  • Physics allows objects to interact like in real world
  • Break the rules of physics!
  • Bodies - defines object type, gravity, mass, elasticity, friction etc.
    • Rigid body
    • Objects that obey physics (until you mess with them
    • Static body
    • Objects that generally don't move. e.g. walls, floor. Allows collision
    • Kinematic body
    • Manage collisions, but break rules of physics. Used for moving objects (e.g. player, platforms, elevators)

Physics cont'd


  • Collision boundaries - define object shape
    • Collision Shape
    • Collision Polygon (Convex and Concave)
  • Gravity & Forces
